# Security Policy

## Reporting Security Issues

This repository contains a demonstration-only representation of selected concepts from a
dual-channel consensus enforcement architecture. It is not a production system and is not
intended for penetration testing, fuzzing, or active security probing.

If you believe you have identified a security-related concern within the demonstration
materials, please report it privately to:

security@enterprisecorporated.com

Public disclosure of potential vulnerabilities is discouraged. Please report any concerns privately via email.

---

## Responsible Disclosure Expectations

- All security reports must be submitted privately via email.
- Public discussion or disclosure of potential vulnerabilities is discouraged.
- This repository does not accept security patches, pull requests, or external code contributions.

---

## Scope

This policy applies only to the demonstration files contained in this repository,
including:

- `demo/` directory
- `example_output.json`
- `fail_output.json`
- `demo_run.md`
- `README.md`

The underlying patented system, including its enforcement logic, cryptographic
mechanisms, validation layers, and operational architecture, is not part of this
repository and is not available for testing or review.

---

## Out-of-Scope

The following are explicitly out of scope:

- Penetration testing or probing of any kind
- Attempts to reverse-engineer or reconstruct the patented system
- Attempts to bypass, modify, or simulate enforcement logic
- Any activity targeting infrastructure, services, or systems outside this repository

---

## Licensing and Access

Access to full system capabilities, including enforcement and validation layers, is
restricted to licensed or approved environments. No rights to the underlying invention,
its methods, or its components are granted through this repository.
